Static Cluster Planning - GigaSpaces Browser


Section Summary: Planning a static cluster before creating it in the GigaSpaces Browser.

General Considerations

Before you begin to create and configure the cluster, consider the following:

  • How many space instances are part of the cluster? On which servers/containers do they run? How should clients and other spaces locate them (LUS/JNDI)?
  • Do you need to set replication links between some of the spaces? Can you identify isolated groups of spaces that require replication?
  • For a specific replication group:
    • Do you want all the entries to be replicated (full replication) or only some of the entries (partial replication)?
    • Do you want all spaces in the group to replicate to all spaces, or do you want to limit replication to specific paths (using a replication matrix)?
    • Do you want notification templates to be replicated?
    • Do you want replicated entries to cause matching notification templates to trigger remote events?
    • In case of a space failure, do you want your clients to failover automatically to another space? Which space should act as the slave space?
    • Do you want to distribute client operations between cluster members? Should there be any restrictions on which operations should be directed to which space?
