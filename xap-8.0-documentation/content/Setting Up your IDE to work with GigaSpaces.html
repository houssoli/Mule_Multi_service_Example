Setting Up your IDE to work with GigaSpaces


Summary: Setting up your IDE to Work With GigaSpaces


Follow these steps to prepare your development environment:

  1. Ensure you have a JDK installed - you will need version 1.5 or higher, latest Java 1.6 is recommended.
    Expand detailed instructions...

    Checking your JDK version

    To check your installed Java version:

    1. Open a command line window.
    2. Run set JAVA_HOME
    3. A response similar to this suggests you have a JDK installed: 
      JAVA_HOME=C:\jdk1.5
    4. To check the JDK version, run %JAVA_HOME%\bin\java -version
      A response like this from Java indicates you have a good JDK installed: 
      java version "1.6.0_23"
Java(TM) SE Runtime Environment (build 1.6.0_23-b05)
Java HotSpot(TM) Client VM (build 11.0-b16, mixed mode, sharing)

      If your installed JDK version is lower then 1.5 or none is installed, see below on how to install one.

    Installing a proper JDK (Java Development Kit)

    • To install JDK 1.5, download and install JDK 5.0 Update X
    • To install JDK 1.6, download and install JDK 6 Update X

  4. Optional - If you're not connected to the internet during development, and want code completion, configure your IDE for XML auto-complete when off-line.
    Expand detailed instructions...

    Adding OpenSpaces XML support to your IDE

    Eclipse 3.5

    The following steps adds auto-complete support to your XML editor:

    1. In Eclipse, select Window > Preferences... to open the Preferences window
    2. On the right side of the window, select XML > XML Catalog
    3. Click the Add.. button, to open the Add/XML catalog entry dialog
    4. In the Location field, click the folder icon and select File System...
    5. Browse to <GigaSpaces root>/lib/optional/openspaces/schema folder
    6. Select one of the xsd files
    7. Click OK to add the entry
    8. Add all 9 xsd files the same way
